# 在哪里执行数据导入操作?

根据您选择的导入方法,执行位置和方式不同。

---

## 🌐 方法一: Web界面导入 (最简单,推荐新手)

### 在哪里执行?
**在您的浏览器中操作**

### 具体步骤:

1. **打开应用**
   ```
   访问: https://3000-it8w3kql0c9adrgr2h8lu-c5f95d36.us2.manus.computer/admin/import
   ```

2. **准备数据文件**
   - 在您的**本地电脑**上创建JSON文件
   - 例如在桌面创建 `vocabulary.json`:
   ```json
   [
     {"expression": "会う", "reading": "あう", "meaning": "见面", "level": "N4"},
     {"expression": "青い", "reading": "あおい", "meaning": "蓝色的", "level": "N4"}
   ]
   ```

3. **上传并导入**
   - 点击"选择文件"按钮
   - 选择您本地的JSON文件
   - 点击"开始导入"
   - 等待导入完成,查看结果

### 优点:
- ✅ 无需命令行知识
- ✅ 可视化操作
- ✅ 实时查看进度
- ✅ 自动错误提示

## 💻 方法二: 命令行脚本 (推荐大批量)

### 在哪里执行?
**有两种方式:**

### 方式A: 在Manus对话中执行 (最简单)

1. **准备数据**
   - 告诉我您的数据格式
   - 我会帮您创建导入脚本

2. **我来执行**
   - 我会在沙箱环境中运行脚本
   - 实时显示导入进度
   - 您只需等待结果

**示例对话**:
```
您: 我有1000个N4词汇要导入,数据格式是...
我: 好的,我来创建脚本并执行导入
```

### 方式B: 在您自己的电脑上执行

**前提条件**:
- 已安装Node.js (v18+)
- 已克隆项目到本地

**步骤**:

1. **克隆项目到本地**
   ```bash
   # 在您的电脑终端(Mac/Linux)或命令提示符(Windows)中执行:
   git clone https://github.com/your-username/japanese-learning-app.git
   cd japanese-learning-app
   ```

2. **安装依赖**
   ```bash
   npm install -g pnpm
   pnpm install
   ```

3. **配置数据库连接**
   - 创建 `.env` 文件
   - 添加数据库URL:
   ```
   DATABASE_URL=mysql://vYSQoPEMjLoro3N.root:your_password@gateway03.us-east-1.prod.aws.tidbcloud.com:4000/W7xsTD72zLTLxKgC6bEdsv
   ```

4. **准备数据文件**
   ```bash
   # 创建数据目录
   mkdir -p data
   
   # 将您的数据保存为 data/vocabulary.json
   ```

5. **修改导入脚本**
   - 编辑 `scripts/direct-import-jlpt-data.mjs`
   - 将数据源改为您的文件

6. **执行导入**
   ```bash
   node scripts/direct-import-jlpt-data.mjs
   ```

## ❓ 常见问题

### Q: 我没有编程基础,能导入数据吗?
**A**: 可以!使用Web界面(/admin/import),只需:
1. 在记事本创建JSON文件
2. 在浏览器上传文件
3. 点击导入按钮

### Q: 我的数据在Excel表格里,怎么办?
**A**: 两种方式:
1. 另存为CSV格式,直接上传到Web界面
2. 发给我,我帮您转换并导入

### Q: 导入会覆盖现有数据吗?
**A**: 不会!脚本会自动检查重复,跳过已存在的词汇。

### Q: 导入失败了怎么办?
**A**: 
1. 查看错误信息
2. 检查数据格式是否正确
3. 告诉我错误信息,我帮您解决

### Q: 可以导入多少数据?
**A**: 理论上无限制,但建议:
- Web界面: 1万条以内
- 命令行脚本: 10万条以内
- 分批导入: 超过10万条
